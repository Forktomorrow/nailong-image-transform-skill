# 验收清单

## 基线保护

- 页面任务、信息架构和核心流程与过滤前一致。
- 所有链接、按钮、输入、菜单和弹层仍可操作。
- 键盘焦点可见，语义标签和替代文字完整。
- 正文对比度至少 4.5:1，大字至少 3:1。
- 手机端触控区不小于 44×44px，布局无横向溢出。
- 数据、单位、图例、错误信息和法律文本未被奶龙文案改写。

## 奶龙系统

- 能说清本次五轴值，实际覆盖与配置一致。
- 宏观、中观、微观层使用同一角色语法，不是多个无关贴纸。
- 至少一个主要元素承担明确的奶龙叙事角色。
- C3–C4 时，奶龙化跨越至少三个设计层：Token、结构、组件、图片、动效中的三个。
- R1–R3 时能看到清晰尺度层级和密度衰减，保留足够负空间。
- 小尺寸图形使用抽象签名，完整角色面孔不过量。

## 性能与动效

- 优先使用 CSS Token、伪元素、背景图和少量 DOM；避免为分形填充创建数百节点。
- 动效使用 transform/opacity，滚动和交互时无明显掉帧。
- `prefers-reduced-motion` 下移除持续运动和大幅位移。
- 图片具备合理尺寸、懒加载策略和稳定的宽高比。

## 回退规则

- 若功能受损，先回退结构层，保留配色和图片层。
- 若视觉噪声过高，降低 R，再降低 C；不要直接删除统一 Token。
- 若品牌识别下降，提高 P，并让奶龙色彩服从品牌系统。
- 若角色不清楚，减少微观元素，强化一个宏观奶龙轮廓。
