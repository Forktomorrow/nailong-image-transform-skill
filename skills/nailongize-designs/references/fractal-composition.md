# 分形与多尺度奶龙填充

这里的“分形”指递归的尺度组织，不要求严格数学自相似。目标是让大奶龙、小奶龙和抽象奶龙符号在同一流场中形成层级，而不是随机平铺。

## 生成流程

1. **定义遮罩**：确定允许奶龙化的不规则区域；排除文字、表单、导航、数据和必要留白。
2. **提取骨架**：判断区域的中心线、主旋涡、边界转折和视觉重心。
3. **放置宏观锚点**：用 1–3 个大奶龙建立整体轮廓或方向。大角色承担语义，不承担填满所有空隙。
4. **中尺度跟随**：沿骨架、边界和大角色周围放置中型奶龙，朝向服从同一流线。大奶龙身边接小奶龙时，要形成亲子、跟随、环抱或群落关系。
5. **微尺度补隙**：剩余小空隙使用眼点、肚皮椭圆、尾巴卷、短爪印或极简小奶龙。尺度不足以画清四肢时，禁止硬画完整角色。
6. **衰减密度**：从视觉焦点向外逐渐减小尺寸、对比度和细节；保留 20%–40% 负空间。
7. **统一媒介**：所有尺度共享同一笔触、光照、像素网格、材质或 CSS Token。

## 递归限制

- 前端界面：默认 `R1`，最高 `R2`。
- 大幅海报、背景、长卷：可用 `R3`。
- 小于 24px 的图形只使用抽象签名，不使用完整奶龙。
- 同一局部最多出现 3 个明显面孔，避免“眼睛噪声”。
- 不让小奶龙覆盖大奶龙的脸、肚皮分区或动作关节。

## 不规则图形填充策略

- **长条/螺旋**：一只大长条奶龙沿中心线盘绕，中小奶龙沿外缘同向跟随；不要横向切断路径。
- **团块/岛屿**：大奶龙确定体块，中型奶龙填边界凹口，小符号填内部纹理。
- **树状/分支**：大奶龙作主干，尾巴或小奶龙群沿分支生长；保持分支层级。
- **环形**：头尾形成闭合或近闭合关系，小奶龙沿圆周保持同向，不挤占中心留白。
- **纹理场**：使用抽象眼点、肚皮斑块和尾巴卷纹，不重复完整角色照片。

## 前端实现选择

- 关键控件：不要使用分形填充。
- 纯装饰背景：可用 CSS mask、background pattern、canvas 或预生成位图。
- 复杂角色群落：优先生成独立图片资产并响应式裁切。
- 数据可视化：只在不改变数据含义时使用多尺度标记；否则限制为色彩和线型。

## 失败诊断

- 看起来像壁纸：缺少宏观锚点和尺度差异。
- 看起来像角色堆积：缺少流线、亲疏关系和负空间。
- 大奶龙不清楚：小尺度元素覆盖了脸、肚皮或主轮廓。
- 界面不可用：递归区域侵入了受保护层。
- 性能下降：DOM 中放置了过多节点；改为单张位图、CSS 背景或 canvas。
