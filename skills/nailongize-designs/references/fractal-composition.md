# 分型拟合与多尺度奶龙构图

这里的“分型”首先指多实例、多尺度的形态拟合，不是图像嵌套，也不是严格数学分形。目标是用若干同画风的大、中、小奶龙共同逼近一个不规则轮廓：大角色建立构图，中角色补凹角，小角色补窄缝，同时维持合理的群体关系和负空间。

## 核心区别

- 正确：多只独立奶龙以不同姿态和尺度共同拟合一个区域。
- 正确：组件自身变成奶龙形态，例如躺卧按钮、蜷曲侧栏、探头标签。
- 错误：在一个奶龙图片里嵌套另一张奶龙图片。
- 错误：把不同名画、照片或水墨风格的奶龙缩略图塞进同一个界面。
- 错误：只增加黄色面积、圆角和黑色边框，却没有头、肚皮、尾巴、短肢、姿态或表情等角色结构。

## 生成流程

1. **定义遮罩**：确定允许奶龙化的不规则区域；排除文字、表单、导航、数据和必要留白。
2. **提取骨架**：判断区域的中心线、主旋涡、边界转折和视觉重心。
3. **生成候选姿态**：为每个尺度准备躺卧、坐立、蜷曲、伸展、探头、奔跑等候选形态，按区域几何选择，而不是把同一姿态等比复制。
4. **放置宏观锚点**：用 1–3 个大奶龙建立整体轮廓、重心和叙事方向。大角色不负责填满全部边角。
5. **中尺度补形**：沿凹角、边界转折和大角色周围放置中型奶龙；允许多个大/中角色共同拟合复杂轮廓。
6. **微尺度补隙**：剩余窄缝使用小奶龙或眼点、肚皮椭圆、尾巴卷等抽象签名。尺度不足时减少细节，不要硬塞完整五官。
7. **构图求解**：控制碰撞、朝向、视线、动作因果和负空间。角色之间形成跟随、环抱、托举、排队或对望关系，不形成随机堆积。
8. **衰减密度**：从视觉焦点向外逐渐减小尺寸、对比度和细节；保留 20%–40% 负空间。
9. **统一媒介**：所有尺度共享同一笔触、光照、像素网格、材质或 CSS Token。

## 递归限制

- 前端界面：默认 `R1`，高强度使用 `R2`。`R2` 指大、中、小三个可见尺度，不指素材嵌套。
- 大幅海报、背景、长卷：可用 `R3`。
- 小于 24px 的图形只使用抽象签名，不使用完整奶龙。
- 同一局部最多出现 3 个明显面孔，避免“眼睛噪声”。
- 不让小奶龙覆盖大奶龙的脸、肚皮分区或动作关节。

## 不规则图形填充策略

- **长条/螺旋**：优先一只长条奶龙沿中心线盘绕；若边缘仍有难以拟合的凹口，再用同向的中小奶龙补形，不横向切断路径。
- **团块/岛屿**：大奶龙确定体块，中型奶龙填边界凹口，小符号填内部纹理。
- **树状/分支**：大奶龙作主干，尾巴或小奶龙群沿分支生长；保持分支层级。
- **环形**：头尾形成闭合或近闭合关系，小奶龙沿圆周保持同向，不挤占中心留白。
- **纹理场**：使用抽象眼点、肚皮斑块和尾巴卷纹，不重复完整角色照片。

## 前端实现选择

- 关键控件：不要在文字命中区堆叠角色，但允许控件外壳直接成为单只躺卧、蜷曲或探头奶龙；语义与 44px 命中区保持规则。
- 纯装饰背景：可用 CSS mask、background pattern、canvas 或预生成位图。
- 复杂角色群落：优先生成独立图片资产并响应式裁切。
- 数据可视化：只在不改变数据含义时使用多尺度标记；否则限制为色彩和线型。

## 失败诊断

- 看起来像壁纸：缺少宏观锚点和尺度差异。
- 看起来像角色堆积：缺少流线、亲疏关系和负空间。
- 看起来像拼贴：混用了不同画风、不同媒介或其他作品的奶龙素材。
- 看起来只是黄色主题：组件没有获得可辨认的头、肚皮、尾巴、短肢、姿态和表情。
- 大奶龙不清楚：小尺度元素覆盖了脸、肚皮或主轮廓。
- 界面不可用：递归区域侵入了受保护层。
- 性能下降：DOM 中放置了过多节点；改为单张位图、CSS 背景或 canvas。
