# 前端层级转换矩阵

| 层级 | 奶龙化方式 | 高强度做法 | 必须保护 |
|---|---|---|---|
| 全局 Token | 奶油黄、肚皮米白、墨黑、低饱和环境色；身体圆角与尾巴曲线 | 全部语义色映射到奶龙色系，并保留成功/警告/错误差异 | WCAG 对比度、状态色含义 |
| 页面骨架 | 梨形重心、蜷缩圆弧、长尾流线、窥视式边缘露出 | hero、分区和滚动叙事形成一只大奶龙或群落的宏观轮廓 | 阅读顺序、响应式、内容容量 |
| 导航 | 使用一个奶龙标记、肚皮底板或尾巴路径指示当前位置 | 导航沿柔和弧线组织，但链接仍保持规则命中区 | 键盘导航、44px 触控区、当前位置状态 |
| 卡片 | 肚皮色内容区、梨形比例、局部耳/尾轮廓 | 不同卡片承担头、身体、尾巴等群体关系 | 文本对齐、点击范围、可扫描性 |
| 按钮 | 肚皮胶囊、短促压缩回弹、单个眼点提示 | 主要按钮可像奶龙肚皮，次要按钮保持安静 | 标签清晰、disabled/focus/hover 状态 |
| 表单 | 奶油底、黄色焦点环、温和圆角 | 只在空状态和成功反馈出现角色 | 原生语义、错误信息、输入效率 |
| 图标 | 尾巴卷、眼点、短爪等抽象签名 | 小尺寸优先抽象符号，不画完整角色 | 图标语义、替代文字 |
| 图表 | 奶龙色板、尾巴曲线、肚皮面积形状 | 允许点位变为小奶龙，但数据密度高时只改色和线型 | 数值、比例、图例、色盲可读性 |
| 图片 | 重要对象按形态矩阵变成奶龙 | 调用图片 Skill 重绘，而不是叠贴纸 | 构图、文字、版权、水印 |
| 空状态 | 一只完整奶龙承担解释与情绪 | 根据错误、等待、完成状态切换姿势 | 下一步操作必须明显 |
| 加载 | 尾巴环绕、肚皮呼吸、眼睛眨动 | 仅在等待时间可感知时出现 | 性能、减少动效、不可阻塞 |
| 动效 | 呼吸、蹒跚、弹肚皮、摇尾 | 用一次编排好的运动系统，不散落随机弹跳 | 60fps、prefers-reduced-motion |
| 文案 | 少量“奶龙化”语气或状态命名 | C4 可改微文案，但不改业务术语和法律文本 | 准确性、品牌语气、国际化 |

## 结构规则

- 先用 Token 和比例建立统一性，再添加角色图像。
- 每个屏幕只选择 1 个主要奶龙叙事，不让所有组件争抢注意力。
- 高强度不等于高噪声。C4 应增加跨层一致性，而不是增加贴纸数量。
- 小尺寸组件优先使用抽象签名；完整奶龙留给 hero、空状态、插画和关键反馈。
- 真实产品品牌存在时，让奶龙系统服从品牌色与排版，不覆盖官方标志。
