# 跨模型通用提示词

不支持 Codex Skill 的生图平台，上传原图后粘贴下面的指令。先把尖括号内容替换成对当前图片的判断。

若平台支持多图参考，同时上传 `../assets/reference/nailong-reference-board.webp`。参考板用于锁定角色身份，原图用于锁定构图与画风；两者职责不同，不能只上传其中一张。

## 通用编辑块

```text
请把输入图像作为需要精确编辑的底图。先识别画面中决定主题、视觉焦点或笑点的关键元素，再把这些元素逐一替换成奶龙；不要把整张图简单染黄，也不要在空白处随机增加奶龙。

优先替换：<主角/最大物体/核心道具>。
其次替换：<月亮、太阳、重复图标或其他视觉锚点>。

先判断对象属于哪种形态：紧凑圆形、长路径、地形块、动态主体或重复阵列。每个奶龙实例必须继承原元素的位置、大小、朝向、动作、透视、遮挡和画面功能。圆形元素变成紧凑蜷缩奶龙；楼梯、河流或道路变成一只有明确头尾、肚皮和短肢的长条盘绕奶龙；山体采用远看为山、近看为奶龙的双重识别；人物和动物保持原动作、重心与情绪；重复元素保持原数量和节奏。

角色身份必须以随附奶龙参考板为准：黄色圆润矮胖，圆钝大头与胖圆身体连续相接、没有细脖子；胸腹有大而连续的浅色肚皮；手脚短粗圆钝并带小爪点；尾巴根部厚、整体粗短；深色眼睛较大、位置偏高且有高光，小鼻孔和小嘴位于脸部较低处，神态憨、天真、略迟钝。允许比例和姿态适配原元素，但不得退化为黄色椭圆、胶囊、通用小龙、细尾蛇形或只有黑点眼睛的 CSS 吉祥物。

严格保留原图的画幅、镜头、构图、主体间距、背景、色彩关系、光影、笔触或材质、清晰度、噪点和未指定元素。原图文字逐字保留，位置、字号和排版不变。只修改被选中的关键元素。

不要出现：额外角色、满屏复制、黄色涂抹、构图漂移、文字乱码、错误肢体、其他黄色卡通角色、水印或新增边框。
```

## 多尺度拟合块

当目标是不规则区域、网页结构或用户要求 `R0–R3` 分型拟合时，在通用编辑块后追加：

```text
把 R0、R1、R2、R3 视为四次独立的完整构图求解。每次改变 R，都重新分析目标区域的主轴、凹角、窄缝、视觉重心和负空间，并重新决定奶龙的数量、尺度、姿态、朝向、位置与遮挡。不得沿用上一层构图后只追加缩小副本。每一级都必须单独成立；同一级内部的大、中、小奶龙共享同一画风、材质和光线。
```

## 已明确替换对象时

如果模型容易自作主张，改用显式映射：

```text
只执行以下替换：
1. <元素 A> -> <对应姿态的奶龙>；保持 <位置/大小/方向/功能>。
2. <元素 B> -> <对应姿态的奶龙>；保持 <位置/大小/方向/功能>。
除以上对象外，其他内容不变。保留全部文字和原画风。
```

## 平台使用建议

- OpenAI、Gemini/Imagen 等对话式编辑模型：上传原图，先使用“通用编辑块”；若结果漂移，第二轮只写一个局部修正，并重申不变量。
- Midjourney 等偏重重绘的平台：同时提供原图作为图像参考，强调 `same composition`、`same number and placement`、`preserve all captions verbatim`；它更可能重构细节，不适合要求像素级锁定的任务。
- Stable Diffusion、FLUX 或节点式工作流：优先用局部重绘、遮罩或 ControlNet/结构控制。遮罩只覆盖待替换元素，并降低重绘幅度，避免背景被改写。
- 模型不认识“奶龙”时：附上本 Skill 的示例图，并同时保留完整外观描述，不要只写角色名。

## 二次修正模板

```text
仅修正这一项：<问题>。
继续保持原图构图、背景、全部文字、未替换元素，以及现有奶龙的位置和数量不变。不要重新设计整张图。
```
