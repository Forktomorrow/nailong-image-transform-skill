# 防过拟合 Benchmark Protocol

## 目的

Benchmark 不能只由已展示的名画组成，否则算法可能记住“星空、新月、山水”的规则，而没有学会从任意形状生成合适的奶龙拓扑。评测必须把“调参数据”和“最终验收数据”分离。

## 四层数据划分

| 层 | 用途 | 是否允许看分数 |
| --- | --- | --- |
| Train | 姿态库、拓扑模板、风格场参数的开发 | 允许 |
| Dev | 选择算法版本和超参数 | 允许，但只看汇总分 |
| Public validation | 发布前公开回归测试 | 允许，不看单图细节 |
| Hidden holdout | 新图、新形状、新画风和新遮挡 | 不允许在调参时查看 |

划分必须按“形状家族”和“画风家族”隔离，而不是简单随机切图。例如训练集可以包含圆形和长路径，隐藏集必须加入从未出现过的月牙、环形缺口、分叉路径和不对称轮廓；训练集有油画，隐藏集加入水墨、壁画、版画和低分辨率照片。

## 场景扩展

隐藏集至少包含：

- **几何变化**：圆、月牙、环、星形、分叉、窄缝、长曲线、非凸块、孔洞；
- **姿态变化**：静态、蜷缩、伸展、奔跑、飞行、抱持、接触；
- **媒介变化**：油画、水墨、壁画、木刻、像素画、照片；
- **扰动**：旋转、缩放、透视、局部遮挡、低对比度、噪声、压缩、色偏；
- **布局变化**：边缘目标、部分出框、前后景遮挡、重复阵列和不同留白。

名画案例只作为语义验收集，不能作为唯一测试集。应额外生成不带艺术家/作品标签的程序化形状和授权照片，用来检验算法是否依赖画名或固定构图。

## 自动指标

1. `mask IoU`：目标区域覆盖是否准确。
2. `boundary F-score`：新奶龙轮廓是否贴合目标几何。
3. `topology completeness`：头身、肚皮、四肢、尾巴和面部锚点是否完整。
4. `style consistency`：目标区域与周围环带的亮度、色彩、梯度方向和纹理尺度差异。
5. `background preservation`：非目标区域的 SSIM/PSNR 或像素差。
6. `seam score`：边缘窄带的梯度跳变、白边和景深不连续。
7. `runtime/memory`：单张图耗时、峰值内存和是否需要网络。

自动指标只能做筛选，最终仍要做盲评。任何背景漂移、硬边或固定站立奶龙都触发失败门槛。

## 综合分数

```text
score = 0.15 * recognition
      + 0.20 * topology
      + 0.15 * composition
      + 0.20 * style
      + 0.15 * fusion
      + 0.10 * background_preservation
      + 0.05 * speed
```

模型/算法版本只能根据 Dev 分数选择；最终报告必须同时给出 Public validation 和 Hidden holdout，不得只报最高的一张图。

## 当前状态

当前仓库只有程序版《星空》和程序版人像的输出，尚未覆盖完整隐藏集。因此目前只能报告：`starry-raster-p2 = 67/100`，其余案例为“未测”，不能推导出通用算法分数。
