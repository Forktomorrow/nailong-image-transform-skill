# 无模型算法的上限与优化路线

## 先定义“最高能做到什么样”

不调用生图模型时，算法只能重排和重建已有像素/矢量信息，不能凭空推断真实遮挡后的新笔触。因此上限取决于目标物体的复杂度：

| 场景 | 现实上限 | 原因 |
| --- | ---: | --- |
| 网页组件、按钮、卡片、徽标 | 85–95/100 | 结构清晰，遮挡少，可直接矢量重建 |
| 旋转楼梯、河流、丝带、云团 | 75–88/100 | 中轴线明确，但需要连续曲线变形 |
| 单个简单插画物体 | 70–82/100 | 可继承局部笔触和光照 |
| 《星空》中的星体/旋涡 | 65–78/100 | 笔触方向复杂，需做纹理场重采样 |
| 蒙娜丽莎式人物、真实人像 | 50–70/100 | 姿态、遮挡、脸部结构和材质变化多 |
| 多人物、复杂前后景名画 | 40–62/100 | 需要背景补全和语义重绘，纯经典算法难以可靠推断 |

这些是工程上限估计，不是当前实现分数。当前程序版《星空》49 分、人像 26 分，说明还处在“贴图/布局原型”阶段。

## 提分顺序

### P0：反贴图硬门槛

- 新输出必须使用奶龙拓扑生成的 mask，禁止把参考 PNG 的透明边界直接作为结果边界。
- 计算边缘色差、alpha 白边、局部锐度和景深差；任一超过阈值直接判定为失败。
- 参考 PNG 只能提供拓扑和姿态，不能提供最终纹理。

### P1：原画风格场

- 在原目标 mask 内估计低频光照场 `L`、颜色场 `C` 和笔触方向场 `G`。
- 奶龙模板的每个点用 `(弧长、法向距离)` 关联到原物体采样坐标。
- 将 `L/C/G` 重采样到新 mask，令新奶龙保持原物体的媒介特征。
- 油画使用方向性笔触核；水墨使用边缘扩散和墨色浓度；照片使用局部景深和噪声统计。

### P2：梯度域融合

- 目标区域内部用原画材质填充；边缘窄带用 Poisson 或多频带融合。
- 只优化窄边带，不允许修改冻结背景。
- 对遮挡缺口使用邻域 patch synthesis，而不是整图重绘。

### P3：迭代优化

每次只优化一个对象，目标函数为：

```text
E = 0.25 * mask_fit
  + 0.20 * topology
  + 0.20 * style_field
  + 0.20 * edge_fusion
  + 0.15 * composition_lock
```

采用“先几何、后材质、最后边缘”的坐标下降；若风格或融合分数没有提升，就回滚，不通过增加奶龙数量掩盖问题。

## 新的发布门槛

- `>= 80`：可用于规则图形和网页设计展示。
- `70–79`：可用于简单插画和部分路径物体。
- `60–69`：仅限实验性名画预览。
- `< 60`：不得展示为最终奶龙化结果。
- 风格保持或融合任一项为 0：无条件拒绝。
