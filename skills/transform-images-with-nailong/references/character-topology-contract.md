# 奶龙角色拓扑合同

生成或改造任何奶龙图像、组件和多尺度构图前，先读取本文件，并同时查看 `../assets/reference/nailong-reference-board.webp`。

## 允许变化

- 为适配按钮、侧栏、山体、旋涡和不规则遮罩，可以拉伸身体比例、改变姿态、朝向、尺度、材质与画风。
- 可以让角色继承油画、水墨、壁画、照片或网页的光线与表面质感。
- 可以省略不影响当前视角的局部，例如被遮挡的脚、尾端或背棘。

## 不可丢失的识别拓扑

1. **头身连续**：圆钝大头和胖圆躯干近乎连成一体，颈部很弱。禁止画成“圆头接细脖子”。
2. **胖圆体块**：正面轮廓由脸、肩、鼓肚和短腿形成连续的大体块。禁止退化为胶囊、豆子或普通椭圆。
3. **浅色大肚皮**：胸腹部必须保留一块明显、连续的奶白或浅黄色椭圆区域。
4. **短粗四肢**：手脚短、厚、圆，脚趾或爪点小而深色。禁止细长手臂、火柴腿或鸟翅。
5. **粗短尾巴**：尾巴从臀部自然长出，根部厚，长度克制。禁止细长蛇尾。
6. **面部锚点**：大而有高光的深色眼睛位于脸部较高位置；鼻孔和小嘴位于较低位置，表情憨、天真、略迟钝。
7. **龙类提示**：允许出现克制的小背棘或头顶小棘，但不能破坏圆钝主轮廓。

## 抽象强度边界

- 最小可接受抽象仍需同时保留：头身连续轮廓、浅色肚皮、至少一只短肢、粗尾或背棘、眼鼻嘴关系。
- 只有黄色、黑点眼睛、奶白椭圆和圆角外框，不构成奶龙。
- 只有“头 + 肚皮 + 细尾巴”的 CSS 拼图，不构成奶龙。

## 逐级重构规则

`R0`、`R1`、`R2`、`R3` 是四次独立求解，不是增量叠加：

1. 每次改变 R，都重新分析目标遮罩的主轴、凹角、窄缝、视觉重心和安全区。
2. 重新选择角色数量、尺度、姿态、朝向、遮挡关系和负空间。
3. 大角色不必保留上一层的位置；中小角色也不能只是追加到上一层空白处。
4. 每一级都必须输出一张自身成立的完整构图。
5. 只有同一构图内部的大、中、小实例共享画风与光线；禁止图片套图片。

## 失败判定

- 角色换成任意黄色吉祥物仍不影响画面：失败。
- 去掉肚皮或短粗四肢后仍看不出差别：失败。
- 调高 R 只多出若干缩小副本，主体位置和负空间完全不变：失败。
- 组件只换黄色、黑边和圆角，没有角色身体结构：失败。
