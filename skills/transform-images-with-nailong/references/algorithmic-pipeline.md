# 不依赖生图模型的拟合算法

卷积和池化本身不是生成器：它们可以提取边缘、纹理和语义特征，但不能凭空重建奶龙的头、肚皮、四肢和尾巴。更稳的无模型方案是“轻量视觉分析 + 奶龙资产几何拟合 + 遮罩合成”。

## 推荐流水线

1. **对象提取**：优先使用用户提供的 mask；否则用边缘、颜色聚类、显著性和连通域得到候选对象。需要语义识别时可选本地视觉模型，但不把它和生成模型混为一谈。
2. **形状表示**：对候选区域计算轮廓、骨架/中心线、距离变换和主方向。长路径用骨架，块状对象用轮廓，窄缝与凹角用距离变换的局部极小值。
3. **多尺度拟合**：先放置一只大奶龙覆盖主体，再在残余区域放置中奶龙，最后用小奶龙填补凹角和窄缝。每只资产都必须保留头身连续、浅色大肚皮、短粗四肢、粗短尾和高位高光眼；禁止用黄色椭圆或色块代替。
4. **姿态与布局**：沿中心线估计切线方向，旋转资产；用宽度场控制缩放；在急转弯处使用侧卧/蜷缩姿态。头尾落在语义端点，肚皮朝向路径内侧，避免“奶龙素材堆在一起”。
5. **遮罩与合成**：先做前景/背景分层，再按深度顺序合成；保留原图的光照、纹理、笔触和文字。对于原图画风，可只把奶龙资产做颜色、噪声和笔触迁移，不改变拓扑。
6. **验收**：检查覆盖率、头尾可见性、肚皮连续性、姿态方向、原构图和文字。若局部不合格，只重算该对象，不重做整张图。

## 适用边界

该路线非常适合前端控件、卡片、徽标、路径、旋转楼梯和规则插画；对复杂名画、真实人物、自然遮挡和全局画风重绘只能作为预处理或低成本预览。最实用的混合方案是：算法负责“找哪里、怎么拟合、生成 mask”，模型负责“把拟合结果融入原媒介”。
